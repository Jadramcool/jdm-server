### 3.2 ✅ Token机制 - 双Token实现

#### 架构设计

采用 **Access Token + Refresh Token** 双Token机制：

| Token类型     | 用途             | 有效期 | 存储位置            |
| ------------- | ---------------- | ------ | ------------------- |
| Access Token  | API请求认证      | 30分钟 | 内存 + localStorage |
| Refresh Token | 刷新Access Token | 7天    | localStorage        |

#### 后端实现

**1. JWT类扩展** (`src/jwt/index.ts`)

```typescript
// 创建Access Token (短期)
public createAccessToken(data: object) {
  return jsonwebtoken.sign(data, this.secret, {
    expiresIn: this.accessTokenExpire
  });
}

// 创建Refresh Token (长期)
public createRefreshToken(data: object) {
  return jsonwebtoken.sign(data, this.secret, {
    expiresIn: this.refreshTokenExpire
  });
}

// 验证Refresh Token
public verifyRefreshToken(token: string) {
  return jsonwebtoken.verify(token, this.secret);
}
```

**2. 登录接口返回双Token** (`src/modules/user/services.ts`)

```typescript
const accessToken = this.JWT.createAccessToken(tokenData);
const refreshToken = this.JWT.createRefreshToken(tokenData);

return {
  data: {
    ...result,
    accessToken,
    refreshToken,
  },
  code: 200,
  message: "登录成功",
};
```

**3. 刷新Token接口** (`src/modules/user/controller.ts`)

```typescript
@Post("/refresh")
public async refreshToken(req: Request, res: Response) {
  const { refreshToken } = req.body;

  try {
    const decoded = this.JWT.verifyRefreshToken(refreshToken);
    const accessToken = this.JWT.createAccessToken({
      id: decoded.id,
      username: decoded.username,
    });
    res.sendResult({ accessToken }, 200, "刷新成功");
  } catch (error) {
    res.sendResult(null, 401, "Token已过期，请重新登录");
  }
}
```

#### 前端实现

**1. Token工具类** (`src/utils/token/index.ts`)

```typescript
// 获取Refresh Token
export const getRefreshToken = () => {
  return lStorage.getItem(refreshTokenKey);
};

// 存储Refresh Token
export const setRefreshToken = (token: string) => {
  lStorage.setItem(refreshTokenKey, token);
};

// 移除Refresh Token
export const removeRefreshToken = () => {
  lStorage.removeItem(refreshTokenKey);
};
```

**2. Auth Store支持双Token** (`src/store/modules/auth.ts`)

```typescript
setToken({ accessToken, refreshToken, token }: any) {
  if (accessToken) {
    this.token = accessToken;
    setTokenLocal(accessToken);
  } else if (token) {
    this.token = token;
    setTokenLocal(token);
  }
  if (refreshToken) {
    setRefreshToken(refreshToken);
  }
}
```

**3. Axios 401自动刷新机制** (`src/utils/http/axios.ts`)

核心逻辑：

- 拦截 401 响应
- 检查是否存在 Refresh Token
- 如果正在刷新中，将请求加入队列等待
- 调用刷新接口获取新 Access Token
- 重新执行失败的请求
- 刷新失败则跳转登录页

```typescript
// 401 处理 - 尝试刷新Token
if (status === 401) {
  const refreshToken = getRefreshToken();

  // 没有refreshToken，直接跳转登录
  if (!refreshToken) {
    handleUnauthorized();
    return Promise.reject(new Error("登录已过期"));
  }

  // 如果正在刷新token，把当前请求加入队列
  if (isRefreshing) {
    return new Promise((resolve) => {
      subscribeTokenRefresh((newToken: string) => {
        config.headers.Authorization = `Bearer ${newToken}`;
        resolve(axios(config));
      });
    });
  }

  // 开始刷新token
  isRefreshing = true;

  try {
    const res = await axios.post(`/user/refresh`, { refreshToken });
    const { accessToken } = res.data?.data || {};

    if (accessToken && config) {
      setToken(accessToken);
      notifySubscribers(accessToken);

      // 重试当前请求
      config.headers.Authorization = `Bearer ${accessToken}`;
      return axios(config);
    }
    throw new Error("刷新失败");
  } catch (refreshError) {
    removeRefreshToken();
    handleUnauthorized();
    return Promise.reject(new Error("登录已过期"));
  } finally {
    isRefreshing = false;
  }
}
```

## 四、待处理问题

### 4.1 ⏸️ 后端无API级别权限校验

**现状：**

- 前端使用 `v-auth` 指令控制按钮显示/隐藏
- 后端API没有任何权限校验

**风险：** 用户可以绕过前端直接调用API，拥有全部操作权限

**建议修复方案：**

1. 创建权限中间件
2. 在关键API添加权限校验装饰器

---

### 4.2 ⏸️ 密码可能明文传输

**位置：** `src/modules/user/services.ts`

**风险：** HTTP请求可能被拦截，密码泄露

**建议方案：**

- 前端使用MD5加密密码后再传输
- 或使用HTTPS加密传输

---

### 4.3 ⏸️ 权限粒度不足

**当前权限级别：**

- ✅ 菜单级别（是否显示菜单）
- ✅ 按钮级别（是否显示按钮）
- ❌ 操作级别（增/删/改/查独立控制）
- ❌ 数据级别（部门数据隔离）

**建议扩展Menu表：**

```prisma
model Menu {
  // ... 现有字段
  permissionType String @default("MENU")  // MENU/BUTTON/OPERATION
  dataScope     String @default("ALL")   // ALL/DEPT/SELF
}
```
